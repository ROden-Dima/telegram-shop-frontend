# 📢 Интеграция с Telegram каналом для управления товарами

## 🎯 Цель

Реализовать автоматическую синхронизацию товаров между Telegram каналом и магазином:
- Публикация товара в канале → автоматическое добавление в магазин
- Удаление сообщения из канала → автоматическое удаление товара
- Редактирование сообщения → автоматическое обновление товара

**Chat ID канала:** `-1003271699368`

### Этап 3: Интеграция с API товаров

**Статус:** ⏳ Ожидает

После парсинга сообщения нужно:
1. Найти категорию по названию (если указана)
2. Загрузить фото из Telegram (через Bot API)
3. Создать товар через API: `POST /api/products`

**Псевдокод:**
```typescript
// 1. Парсим сообщение
const parsed = parseProductMessage(telegramMessage);

// 2. Валидируем
if (!validateParsedProduct(parsed)) {
  return { error: 'Invalid product data' };
}

// 3. Находим категорию
const category = await findCategoryByName(parsed.category);

// 4. Загружаем фото
const photoUrl = await downloadPhotoFromTelegram(parsed.photoFileId);

// 5. Создаем товар
const product = await createProduct({
  product_name: parsed.description,
  description: parsed.description,
  price: parsed.price,
  quantity: parsed.quantity || 0,
  category_ids: category ? [category.id] : [],
  photos: [photoUrl],
  user_id: adminUserId
});
```

## 🔧 Технические детали

### Структура базы данных

Нужно добавить поля в таблицу `products`:

```sql
ALTER TABLE products 
ADD COLUMN telegram_message_id BIGINT UNIQUE,
ADD COLUMN telegram_channel_id BIGINT,
ADD COLUMN sync_status ENUM('synced', 'pending', 'error') DEFAULT 'synced',
ADD INDEX idx_telegram_message_id (telegram_message_id);
```

### API Endpoints (нужно создать на бэкенде)

1. **POST /api/telegram/webhook**
   - Получает события от Telegram
   - Обрабатывает сообщения из канала

2. **POST /api/products/from-telegram**
   - Создает товар из данных Telegram сообщения
   - Body: `{ messageId, parsedData, photoFileId }`

3. **PUT /api/products/from-telegram/:messageId**
   - Обновляет товар по message_id

4. **DELETE /api/products/from-telegram/:messageId**
   - Удаляет товар по message_id

## ⚠️ Важные моменты

1. **Бот должен быть администратором канала**
   - Иначе он не будет получать события

2. **Webhook должен быть настроен правильно**
   - URL должен быть доступен из интернета
   - Должен быть HTTPS (Telegram требует)

3. **Обработка ошибок**
   - Если парсинг не удался - логировать и пропустить
   - Если создание товара не удалось - отправить уведомление админу

4. **Безопасность**
   - Проверять что сообщения приходят из нужного канала
   - Проверять подпись webhook (если используется)
